# Session 2026-05-11

## What Was Built or Changed

**Release 1.1.0 shipped** with three major themes:

1. **V2 mailflow verification**: `check-mailflow.ps1` was rewritten with expanded capabilities (153 lines, up from a smaller baseline). The script now handles more complex mail routing scenarios.

2. **CI/build robustness**: The GitHub Actions verify workflow (`verify.yml`) was overhauled from V1 to V2. V1 silently relied on pre-installed Graph modules in the runner image; V2 now explicitly installs every module declared in `StevesScriptorium.psd1`'s RequiredModules list. This eliminates fragility when GitHub refreshes its runner images.

3. **Audit-log integrity**: `offboard-user.ps1` was substantially reworked (124 lines, up from baseline) to ensure audit log consistency during user offboarding—likely adding validation or retry logic around mailbox operations.

4. **Documentation layer**: 
   - A formal `DESIGN.md` (541 lines) was added, capturing architecture and rationale in one place.
   - 18 Architecture Decision Records (ADR-0001 through ADR-0018) were imported into `docs/decisions/`, documenting design choices like the dot-source dispatch model, dual distribution (module + profile), per-script connection patterns, and logging destinations.
   - `CLAUDE.md` was updated to flag that all significant structural decisions now have ADRs and should be consulted before making changes.

5. **Supporting changes**: `get-allusers.ps1` received updates (23 lines touched), and `kill-graph.ps1` was expanded (61 lines). `README.md` was refreshed with 29 lines of changes.

6. **Housekeeping**: `.claude/settings.local.json` (local session state) was deleted and `.claude/` was added to `.gitignore` to prevent agent artifacts leaking into version control.

## Key Decisions

**V1 → V2 CI migration**: The workflow was rewritten because silent brittleness was unacceptable. Instead of assuming Graph modules exist in the runner image, V2 pins and installs them explicitly. This makes CI behavior reproducible and detectable when it breaks.

**ADR-as-design-constraint**: The decision to backfill 18 ADRs and prominently link them in `CLAUDE.md` reflects a deliberate choice: non-obvious patterns and trade-offs must be documented *before* being changed. This trades effort (writing ADRs) for confidence (future sessions won't cargo-cult patterns without understanding "why").

**Graph submodule pinning**: ADR-0022 indicates explicit pinning of `Microsoft.Graph.*` submodules, suggesting a prior incident or fragility with transitive dependency resolution.

## New Patterns Introduced

**Architecture Decision Records as a first-class artifact**: ADRs are now the canonical place to explain *why* a pattern exists. This is new to the project (commit message flags it as an import, not generation). Future sessions must treat ADRs as decision constraints.

**Explicit RequiredModules installation in CI**: V2 of verify.yml embeds the philosophy "test what you declare"—if a module is in the manifest, install it in CI. This is a shift from implicit reliance on runner image contents.
